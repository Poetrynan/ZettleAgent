# Cognitive Load Theory

John Sweller's Cognitive Load Theory (CLT) posits that human working memory has limited capacity, and instructional design should minimize **extraneous cognitive load**.

## Three Types of Cognitive Load

| Type           | Meaning                             | Optimizable?    |
| -------------- | ----------------------------------- | --------------- |
| **Intrinsic**  | Inherent complexity of the material | Cannot reduce   |
| **Extraneous** | Caused by poor instructional design | Should minimize |
| **Germane**    | Effort to build mental schemas      | Should maximize |

## Implications for Knowledge Management

The \[\[Zettelkasten Method]] naturally aligns with CLT:

* **Atomicity** → Reduces intrinsic load per note

* **Linking** → Externalizes germane load into the system

* **Indexing** → Reduces extraneous retrieval load

## Implications for AI Interface Design

* \[\[AI Agent Architecture]] tool calls should be **transparent but non-intrusive**

* \[\[Retrieval-Augmented Generation]] source citations help users **verify rather than memorize**

* \[\[Knowledge Graphs]] visualization makes abstract relationships **concrete**, freeing working memory

## Miller's 7 plus or minus 2

Miller's classic research showed that working memory can process only **7 ± 2 chunks** at a time. This explains why:

* Good notes should be concise

* Good interfaces should be minimal

* \[\[Emergence]] requires external systems to augment cognition
